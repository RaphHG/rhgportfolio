Projets

1. Infrastructure Active Directory, serveur de fichiers et sauvegarde

Mise en place d’un nouveau contrôleur de domaine avec ajout d’un serveur de fichiers et de la sauvegarde automatisée avec Veeam.

  • Installation d’un hyperviseur VMWare ESXi
  • Déploiement d’un Windows Server 2019 intégré au domaine de l’entreprise
  • Création d’une unité d’organisation (OU) de test et d’un utilisateur test
  • Mise en place d’un partage réseau pour les dossiers "Documents" des utilisateurs
  • Redirection automatique du dossier "Documents" vers le serveur
  • Synchronisation des fichiers en ligne/hors-ligne (mode itinérant)
  • Quota de 10 Go par utilisateur appliqué sur leur dossier personnel
  • Script de session : création automatique du dossier utilisateur s’il n’existe pas
  • Nettoyage de l’Active Directory avec suppression des comptes inactifs

2. Déploiement automatique de postes avec WDS/MDT

Mise en place d’un serveur Windows avec les rôles AD, DHCP, WDS (Windows Deployment Services) et MDT (Microsoft Deployment Toolkit).

  • Configuration du déploiement par le réseau via le protocole PXE (Pre-boot eXecution Environment)
  • Création d’une séquence de tâches avec MDT pour automatiser l’installation des postes
  • Déploiement sans intervention manuelle (mode Zero Touch)
  • Les postes déployés rejoignent automatiquement le domaine
  • Paramétrage post-installation complet (utilisateur, nom du poste, configuration réseau)

3. Projet de supervision avec Grafana, InfluxDB, Telegraf et PowerShell

Conception et déploiement d’un système de supervision pour surveiller l’état des services Windows Server (GPO, DHCP, WDS, MDT) en temps réel.

  • Base de données : InfluxDB utilisée comme moteur de stockage de séries temporelles
  • Visualisation : Grafana connecté à InfluxDB pour afficher les métriques en temps réel via des dashboards dynamiques
  • Collecte : Utilisation de Telegraf pour lire des fichiers de logs générés par des scripts PowerShell
  • Scripts PowerShell : Création de scripts personnalisés exécutés périodiquement via le Planificateur de tâches Windows pour :
    • Vérifier l’état du service DHCP (baux attribués)
    • Lister les GPO appliquées sur les machines
    • Contrôler l’état des services WDS et MDT
    • Exporter les données dans des fichiers lisibles par Telegraf (format key=value)
  • Architecture : Le système est hébergé sur une machine Debian avec 7 Go de RAM pour garantir la fluidité de Grafana et InfluxDB
  • Résultat : Visualisation en temps réel des services critiques, détection rapide des anomalies, amélioration de la réactivité de l'administrateur
