Veille technologique – KeeLoader : version piégée de KeePass utilisée pour déployer un ransomware sur VMware ESXi

Contexte

KeePass est un gestionnaire de mots de passe open source. Des cybercriminels ont créé une version modifiée, appelée KeeLoader, qui conserve les fonctions de KeePass mais y ajoute du code malveillant. Cette version est utilisée comme vecteur d’attaque pour infiltrer des réseaux professionnels et chiffrer des serveurs VMware ESXi à l’aide d’un ransomware.

Fonctionnement de KeeLoader

  • Version trojanisée de KeePass distribuée via des sites frauduleux imitant le site officiel (ex. keeppaswrd[.]com).
  • Installation d’un malware Cobalt Strike Beacon qui donne un accès distant au système infecté.
  • Exfiltration de la base de données KeePass contenant les mots de passe de l’utilisateur.
  • Fonction keylogger intégrée pour capturer les frappes clavier.

Chaîne d’attaque vers ESXi

  1. Après l’installation de KeeLoader, l’attaquant cartographie le réseau interne.
  2. Il identifie les serveurs VMware ESXi vulnérables.
  3. Il déploie un ransomware (souvent Black Basta).
  4. Les machines virtuelles sont chiffrées, rendant les services indisponibles.

Recommandations pour un administrateur SISR

  • Télécharger uniquement les logiciels depuis les sites officiels.
  • Vérifier la signature numérique des installateurs.
  • Mettre en place une surveillance réseau pour détecter les connexions suspectes.
  • Appliquer rapidement les mises à jour de sécurité, notamment sur ESXi.
  • Sensibiliser les utilisateurs aux risques liés aux sources de téléchargement non fiables.
